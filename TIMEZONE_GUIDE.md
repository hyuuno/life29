# 🌍 时区功能使用说明

## 时区转换功能

网站会自动将所有时间戳转换为**加州时间（太平洋时间 PT）**显示。

### 为什么需要时区？

- **你在中国**：北京时间 UTC+8
- **她在加州**：太平洋时间 UTC-8（夏令时）或 UTC-7（标准时间）
- **时差**：16小时（冬季）或 15小时（夏季）

### 如何使用

## 📝 添加内容时的时间戳格式

### 格式1：带时区的完整格式（推荐）

**中国时间（北京时间）：**
```json
{
  "timestamp": "2024-12-01T14:30:00+08:00"
}
```

**加州时间（太平洋时间）：**
```json
{
  "timestamp": "2024-12-01T14:30:00-08:00"
}
```

**说明：**
- `+08:00` = 东八区（北京时间）
- `-08:00` = 西八区（太平洋标准时间，冬季）
- `-07:00` = 西七区（太平洋夏令时，夏季）

### 格式2：简化格式（系统会当作UTC时间）

```json
{
  "timestamp": "2024-12-01T14:30:00"
}
```

如果不加时区，系统会按 UTC 时间处理，然后转换到加州时间。

## 🕐 时间戳示例

### 示例1：你在中国发布

**场景**：2024年12月1日下午2:30（北京时间）发布

```json
{
  "id": 1,
  "user": "user1",
  "timestamp": "2024-12-01T14:30:00+08:00",
  "text": "刚吃完午饭",
  "images": []
}
```

**网站显示**：
- 日期：2024.11.30 周六
- 时间：22:30（前一天晚上10:30，加州时间）

### 示例2：她在加州发布

**场景**：2024年12月1日上午10:00（加州时间）发布

```json
{
  "id": 2,
  "user": "user2",
  "timestamp": "2024-12-01T10:00:00-08:00",
  "text": "早上好！",
  "images": []
}
```

**网站显示**：
- 日期：2024.12.01 周日
- 时间：10:00

### 示例3：同一天（加州时间）多次上传

```json
{
  "posts": [
    {
      "id": 1,
      "user": "user1",
      "timestamp": "2024-12-01T22:30:00+08:00",
      "text": "北京时间晚上10:30"
    },
    {
      "id": 2,
      "user": "user1",
      "timestamp": "2024-12-01T14:00:00+08:00",
      "text": "北京时间下午2:00"
    },
    {
      "id": 3,
      "user": "user2",
      "timestamp": "2024-12-01T08:00:00-08:00",

```

**网站显示（按加州时间）：**
- 2024.12.01 周日
  - 06:30 - user1: 北京时间晚上10:30
  - 08:00 - user2: 加州时间早上8:00
- 2024.11.30 周六
  - 22:00 - user1: 北京时间下午2:00

## 🛠️ 获取当前时间戳的方法

### 方法1：在线工具

**中国用户：**
访问：https://tool.lu/timestamp/
- 选择"北京时间"
- 复制格式：`2024-12-01T14:30:00+08:00`

**加州用户：**
- 同样网站，选择"洛杉矶时间"
- 复制格式：`2024-12-01T14:30:00-08:00`

### 方法2：手动构造

**格式：**
```
YYYY-MM-DDTHH:mm:ss+时区
```

**示例：**
- 2024年12月1日下午2点30分（北京）
- 写成：`2024-12-01T14:30:00+08:00`

### 方法3：使用代码（开发者）

**JavaScript（浏览器控制台）：**

```javascript
// 获取当前时间（带时区）
new Date().toISOString()
// 输出：2024-12-01T06:30:00.000Z (UTC时间)

// 获取本地时间字符串
const now = new Date();
const offset = -now.getTimezoneOffset() / 60;
const sign = offset >= 0 ? '+' : '-';
const pad = (n) => String(Math.abs(n)).padStart(2, '0');
const timestamp = `${now.getFullYear()}-${pad(now.getMonth()+1)}-${pad(now.getDate())}T${pad(now.getHours())}:${pad(now.getMinutes())}:${pad(now.getSeconds())}${sign}${pad(offset)}:00`;
console.log(timestamp);
```

**Python：**

```python
from datetime import datetime
import pytz

# 北京时间
beijing_tz = pytz.timezone('Asia/Shanghai')
beijing_time = datetime.now(beijing_tz)
print(beijing_time.isoformat())
# 输出：2024-12-01T14:30:00+08:00

# 加州时间
la_tz = pytz.timezone('America/Los_Angeles')
la_time = datetime.now(la_tz)
print(la_time.isoformat())
# 输出：2024-12-01T22:30:00-08:00
```

## 🎯 最佳实践

### 添加新内容时

1. **始终使用带时区的格式**
   ```json
   "timestamp": "2024-12-01T14:30:00+08:00"
   ```

2. **按你的本地时间记录**
   - 你在中国用 `+08:00`
   - 她在加州用 `-08:00` 或 `-07:00`

3. **不用担心转换**
   - 网站会自动统一转换到加州时间显示
   - 你们看到的时间都是一样的

### 时间线显示规则

- **同一天（加州时间）的多条记录**：
  - 只在第一条显示日期标签
  - 后续记录只显示时间
  
- **日期变化时**：
  - 显示新的日期标签（带星期）
  - 中间用时间轴连接

## ❓ 常见问题

**Q: 我要不要计算时差？**
A: 不需要！直接用你的本地时间，加上时区标识（+08:00），系统自动转换。

**Q: 夏令时怎么办？**
A: 加州夏令时用 `-07:00`，标准时间用 `-08:00`。系统会自动处理。

**Q: 如果我忘记加时区？**
A: 系统会当作UTC时间，可能显示不准确。建议始终加上时区。

**Q: 能改成显示北京时间吗？**
A: 可以！编辑 `script.js` 第7行：
```javascript
targetTimezone: 'Asia/Shanghai'  // 改成北京时间
```
